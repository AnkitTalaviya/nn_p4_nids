# In-Network Intrusion Detection Using Neural Networks and P4 Programming Language

This project follows one common workflow:

1. Open the notebook for the model you want to use.
2. Run all notebook cells to generate the model JSON.
3. Go to the matching runtime folder.
4. Run `./reset_and_run.sh` for single-switch or `./reset_and_run_multi.sh` for multi-switch.

All paths below are relative to the repository root.

## Requirements

### System requirements

- Linux environment
- Python 3
- Jupyter Notebook
- `p4c-bm2-ss` or `p4c`
- BMv2 tools: `simple_switch` and `simple_switch_CLI`
- `sudo`
- `iproute2` for `ip link` commands

### Python packages and setup

Install Python dependencies with:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Data

Download the dataset files from:

`https://cloud.tu-ilmenau.de/s/aZR6Kk6yGdL3mmg`

Place the downloaded files in the repository `data/` folder.

## Common Run Flow

### Single-switch

```bash
cd <single_switch_folder>
./reset_and_run.sh
```

### Multi-switch

```bash
cd <multi_switch_folder>
./reset_and_run_multi.sh
```

## Which Script Generates Which Files

| Script | Input | Main generated files |
| --- | --- | --- |
| `*/single_switch/generate_cmd.py` | model JSON | `commands.txt` |
| `dynamic_runtime_template/scripts/generate_commands.py` | model JSON + selected profile | `commands.txt` |
| `*/single_switch/reset_and_run.sh` | model JSON from notebook | `commands.txt`, compiled BMv2 JSON, final test log, final results CSV |
| `*/multi_switch/generate_multi_cmd.py` | model JSON | `commands_s1.txt`, `commands_s2.txt`, `commands_s3.txt`, `commands_s4.txt` |
| `dynamic_runtime_template/multi_switch/scripts/generate_multi_commands.py` | model JSON + profile + switch count | `commands_s1.txt` ... `commands_sN.txt` |
| `dynamic_runtime_template/multi_switch/reset_and_run_multi.sh` | model JSON + profile + switch count | generated stage P4 files, compiled stage BMv2 JSON files, `commands_s1.txt` ... `commands_sN.txt` |
| `*/multi_switch/reset_and_run_multi.sh` | model JSON from notebook | deploys multi-switch pipeline, final multi-switch log, final multi-switch results CSV |
| `*/single_switch/final_switch_test.py` | running single switch | accuracy report in log and CSV |
| `*/multi_switch/final_multi_switch_accuracy_test.py` | running multi-switch pipeline | accuracy report in log and CSV |

## Notes

- Run the notebook first. The shell scripts expect the model JSON to already exist.
- Single-switch folders use `./reset_and_run.sh`.
- Multi-switch folders use `./reset_and_run_multi.sh`.
- The run scripts compile the P4 program, start BMv2, load commands, and run the final accuracy test automatically.
